# 国家算力网试用过程详细介绍

## 项目背景

**申请单位：** 浙江宁唐健康大数据科技有限公司  
**项目名称：** 宁波医学人工智能创新应用平台  
**试用时间：** 2026年1月 - 至今

---

## 一、需求调研阶段（2025年11月-12月）

### 1.1 初步沟通（2025年11月底）
- **沟通方式：** 线上会议
- **参与方：** 浙江宁唐健康大数据科技有限公司、国家算力网运营团队
- **沟通内容：**
  - 调研公司当前算力使用情况
  - 了解宁波医学人工智能创新应用平台的算力需求
  - 评估现有算力资源与业务发展的匹配度

### 1.2 需求激增与正式对接（2025年12月底）
- **背景：** 随着医疗机构对AI应用及算力资源的需求显著增加，现有算力已无法满足业务扩展需要
- **行动：** 正式启动与国家算力网的对接流程
- **需求确认：**
  - 高性能GPU计算资源（大模型训练与推理）
  - 稳定的网络环境（低延迟内网通信）
  - 弹性扩展能力（应对业务高峰）

---

## 二、资源协调与准备阶段（2026年1月）

### 2.1 多方协调会（2026年1月13日）
- **会议形式：** 线上专题会议
- **参与方：** 
  - 浙江宁唐健康大数据科技有限公司
  - 国家算力网运营团队
  - 电信天翼云技术团队
- **会议成果：**
  - 确定资源配置方案：8卡NVIDIA RTX 4090服务器
  - 明确配套资源需求（存储、网络、安全）
  - 制定部署时间表与技术支持方案
  - 达成首月免费试用合作意向

### 2.2 账号注册与认证（2026年1月19日）
- **完成事项：**
  1. 天翼云平台账号注册
  2. 国家算力网平台账号注册
  3. 企业实名认证（提交营业执照、法人信息等）
  4. 成功领取免费试用优惠券
- **审核状态：** 通过

### 2.3 正式下单与资源验证（2026年1月底）
- **下单流程：**
  1. 登录国家算力网平台
  2. 选择预定配置：8卡4090 GPU服务器集群
  3. 应用免费试用优惠券
  4. 提交资源开通申请

- **资源验证测试：**
  | 测试项目 | 测试内容 | 验证目的 |
  |---------|---------|---------|
  | 内网延迟测试 | 节点间通信延迟 | 确保分布式训练效率 |
  | GPU型号核对 | 确认4090显卡规格 | 验证硬件配置符合需求 |
  | 存储I/O测试 | 读写速度、吞吐量 | 评估数据加载性能 |
  | 网络带宽测试 | 上下行速率 | 确保模型下载/上传顺畅 |

- **试用启动：** 通过验证后，正式进入首月免费试用期

---

## 三、试用期监控指标

在首月试用期间，重点监控以下性能指标：

### 3.1 计算性能指标
- **GPU利用率：** 训练/推理时的GPU占用率
- **显存使用率：** 大模型加载时的显存消耗
- **计算吞吐量：** 每秒处理的样本数或token数
- **多卡并行效率：** 8卡协同工作的加速比

### 3.2 存储性能指标
- **I/O吞吐量：** 数据读取写入速度
- **延迟分布：** 存储访问响应时间
- **缓存命中率：** 高频数据缓存效率

### 3.3 网络性能指标
- **内网延迟：** 节点间通信时延
- **带宽稳定性：** 网络传输速率波动情况
- **丢包率：** 数据传输可靠性

### 3.4 服务稳定性指标
- **可用性：** 服务在线时间百分比
- **故障恢复时间：** 出现问题后的恢复速度
- **技术支持响应：** 工单处理时效

## 四、应用部署与业务验证（2026年2月初）

### 4.1 模型部署
- **部署模型：** Qwen-30B（通义千问30B参数大语言模型）
- **部署环境：** 国家算力网8卡4090服务器
- **优化措施：**
  - 模型量化（降低显存占用）
  - 推理加速（TensorRT/vLLM优化）
  - 并发处理（多请求并行）

### 4.2 医疗数据微调
- **数据来源：** 医院真实诊疗数据（脱敏处理）
- **微调策略：**
  - 基于医疗领域语料的继续预训练
  - 针对诊疗场景的指令微调（Instruction Tuning）
  - 医学知识图谱融合
- **训练配置：**
  - 使用8卡4090进行分布式训练
  - 采用DeepSpeed/Lightning框架加速
  - 学习率调度与梯度累积优化

### 4.3 CDSS场景应用
将微调后的模型应用于**临床辅助决策支持系统（CDSS）**：

| 应用场景 | 功能描述 | 效果提升 |
|---------|---------|---------|
| **诊疗合理性判断** | 分析医嘱、用药、检查的合理性，提示潜在风险 | 准确性提升，误判率降低 |
| **门诊辅助诊疗** | 根据患者症状、病史推荐可能的诊断方向 | 推荐内容更精准、响应速度更快 |
| **住院辅助诊疗** | 针对住院患者的全程诊疗建议，包括用药调整、检查推荐 | 个性化程度提高，符合临床路径 |

### 4.4 性能表现
- **推理速度：** 相比原有环境，响应延迟显著降低
- **推荐准确率：** 医疗数据微调后，CDSS推荐内容的临床相关性提升
- **并发能力：** 8卡4090可同时处理多路诊疗请求，满足医院高峰时段需求

## 五、后续规划

试用期结束后，公司将结合以下因素制定后续算力租用计划：

### 5.1 评估维度
1. **性能表现：** 试用期各项监控指标是否达到预期
2. **成本效益：** 相比自建算力中心或采购硬件的投入产出比
3. **服务质量：** 网络稳定性、技术支持满意度
4. **业务增长：** 医疗机构客户增长带来的算力需求预测

### 5.2 市场因素
- **算力市场价格波动：** 关注GPU租赁市场价格变化
- **政策扶持：** 国家算力网对医疗AI企业的优惠政策
- **竞品对比：** 与其他云服务商（阿里云、华为云等）的性价比比较

### 5.3 决策方向
- **方案A（继续租用）：** 如性能满意且价格合理，签订长期租用合同
- **方案B（混合部署）：** 核心模型训练用国家算力网，边缘推理部署到医院本地
- **方案C（扩容升级）：** 根据业务发展，申请更多算力资源或升级至A100/H100等高端显卡

## 六、经验总结

### 6.1 试用流程亮点
1. **高效对接：** 从需求沟通到资源开通约2个月，响应速度快
2. **多方协作：** 国家算力网、天翼云、企业三方配合顺畅
3. **试用优惠：** 首月免费试用降低了企业试用门槛
4. **技术支持：** 资源验证阶段有专业人员协助测试

### 6.2 关键成功因素
- **明确的需求规划：** 提前梳理好算力规格、网络要求、存储需求
- **充分的验证测试：** 试用初期进行全面性能测试，确保资源符合预期
- **快速的应用落地：** 2月初即完成模型部署和业务验证，证明资源可用性
- **持续监控优化：** 试用期间持续跟踪关键指标，及时发现并解决问题

### 6.3 对医疗AI企业的建议
1. **前期调研充分：** 准确评估AI模型对算力的实际需求（GPU型号、显存、卡数）
2. **重视网络质量：** 医疗数据量大，内网延迟和带宽直接影响训练效率
3. **安全合规优先：** 确保算力平台符合医疗数据安全等级保护要求
4. **弹性扩展规划：** 预留算力扩容空间，应对业务突发增长
